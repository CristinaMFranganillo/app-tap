Enfoque para Estadísticas del Tirador

Dos bloques nuevos en el perfil del socio, reemplazando el gráfico actual

Orden final del perfil del socio

1. Stats (Sesiones / Media / Mejor)
2. Posición en el club
3. NUEVO — Evolución por meses (multi-año)
4. Heatmap esquemas (ya implementado)
5. NUEVO — Heatmap de fallos por plato
A

Enfoque recomendado: Dos features, una spec

Ambas features van en el mismo componente (perfil.component), comparten el anioSeleccionado, y se implementan juntas. Una sola spec cubre ambas.

Pros

  • Coherencia entre features
  • Una sola ronda de review
  • Comparten lógica de filtrado por año

Cons

  • Spec más larga
B

Specs separadas

Una spec por feature: primero evolución por meses, luego heatmap de fallos. Implementación secuencial.

Pros

  • Más enfocadas
  • Menor riesgo por spec

Cons

  • Más overhead de proceso
  • Duplican contexto del componente
